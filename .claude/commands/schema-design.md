## 第 1 步：判断你是否需要 schema

先别急着动手——我们先用 3 个问题确认你是否真的需要 schema 约束。
很多人装了一堆 Zod 校验，结果发现 AI 只是帮他们写文章……完全用不上。

请回答下面 3 个问题（只回答 Y / N）：

**Q1：你要生成的东西，会显示在网页（前端组件）上吗？**
> 例如：AI 生成的产品评测，要在一个 React 页面里分区块展示标题、评分、优缺点。

**Q2：你要生成的东西，会存进数据库吗？**
> 例如：AI 生成的用户画像，要写进 Postgres 的 `profiles` 表，之后还能查询和更新。

**Q3：你要生成的东西，会被其他程序继续处理吗（比如调别的 API、触发后续逻辑）？**
> 例如：AI 生成一个商品推荐列表，要传给支付系统或发邮件服务继续处理。

---

**判断规则：**

✅ 3 个答案里**任意一个是 Y** → 你需要 schema 约束，继续往下走。

❌ 3 个**全部是 N** → 你不需要 schema，用自然语言约束就够了（比如让 AI 写文章、翻译、头脑风暴）。这种情况下，你只需要在 prompt 里说清楚格式要求就行，不必引入 Zod。

---

## 第 2 步：告诉我你要生成什么

用你自己的话描述就好，不需要懂代码，也不需要提 Zod。
告诉我：**这个 AI 要生成什么内容，大概有哪些部分？**

> 示范例子：
> "我想让 AI 生成一份产品分析报告。里面要有：优点 3 到 5 条、缺点 3 到 5 条、市场分析一段话、还有一个竞品对比表格（每行是一个竞品，列出名字、价格区间、核心差异）。"

两种方式都可以：

- **带参数调用**：直接在 `/schema-design` 后面跟上你的描述，例如：
  `/schema-design 用户信息卡片，包含昵称、头像URL、签名、兴趣标签列表（最多5个）`

- **空调用**：直接输入 `/schema-design`，然后我会问你，你再描述也行。

描述收到后，我会直接进入第 3 步，给你完整的 4 件套产出。

## 第 3 步：4 件套产出

这是整个命令最值钱的部分。我会根据你的描述，一次性给你下面 4 样东西，缺一不可。

---

### 产出 1：完整的 Zod schema

每个字段都会带 `.describe()` 中文说明——这段描述会被拼进 AI 的 system prompt，用中文写能让 AI 的输出更贴近中文语境，减少"洋味儿"。

设计原则我会帮你把好关：
- 能枚举的值一定用 `z.enum([...])`，不用 `z.string()` 加后置校验
- 数组字段一定加 `.min()` 和 `.max()`，防止 AI 返回空列表或无限长
- 必填字段不加 `.optional()`，逼 AI 必出；真正可选的才标 `.optional()`

schema 末尾还会附一段 `.parse()` / `.safeParse()` 的使用示例，适用于非流式场景（比如 server action 收到 AI 响应后回验、写数据库前校验）：

```ts
// 非流式路径：收到 AI 响应后校验
const parsed = mySchema.parse(rawJson);

// 推荐写法：用 safeParse 优雅处理错误
const result = mySchema.safeParse(rawJson);
if (!result.success) {
  console.error(result.error.format());
  // 返回错误 or 重试
}
```

---

### 产出 2：配套的 AI 生成提示词

一段可以直接粘进你的 system prompt 的提示词，包含：
- 明确要求"严格遵守以下 schema"
- 把 schema 的 JSON 结构嵌入其中，让 AI 知道每个字段的含义和限制
- 结尾强调"不要返回纯文本，所有内容必须填进 schema 的对应字段"

---

### 产出 3：React 消费代码

使用 Vercel AI SDK 的 `useObject` hook，展示流式接收结构化数据的完整写法：
- 正确处理 `DeepPartial` 类型（流式过程中字段可能还是 `undefined`，需要用 `??` 做兜底）
- 加入骨架屏 / 加载态，让用户体验不跳变

---

### 产出 4：判断力检查

列出 3 到 5 条"你可能没想到但值得再约束的点"，每条说明：

- 原来怎么写（过于宽松）
- 更好怎么写（加约束）
- 为什么这样更好（防止 AI 踩哪些坑）

例如：
> `title: z.string()` → `title: z.string().min(5).max(80)` → 防止 AI 返回空标题，或者一口气写出超长标题塞爆卡片布局

这 4 件套拿到手，你就能直接进项目开写，不用再反复猜 AI 会吐什么格式。
